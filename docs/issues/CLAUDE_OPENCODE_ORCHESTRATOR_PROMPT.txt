OpenCode writer launch pattern:
```bash
mkdir -p docs/execution/opencode-logs docs/execution/opencode-prompts

opencode run \
  --dir "/absolute/path/to/writer-worktree" \
  --format json \
  --title "zentra-issue-<id>-writer" \
  -f "/absolute/path/to/docs/issues/<issue-file>.md" \
  "$(cat docs/execution/opencode-prompts/issue-<id>-writer.txt)" \
  >"docs/execution/opencode-logs/issue-<id>-writer.jsonl" \
  2>"docs/execution/opencode-logs/issue-<id>-writer.stderr" &

```

The prompt path and log path in the launch command must be absolute or rooted in the integration worktree so results are not written into the writer's owned product paths.
