Run serve mode

agent-tail serve run.jsonl
agent-tail serve run.jsonl --port 8765 --open
producer | agent-tail serve -

Files are followed continuously. Standard input is read until EOF, after which the source is disconnected and unterminated traces become incomplete.

agent-tail serve run.jsonl --host 0.0.0.0 --remote-access

Non-loopback hosts require --remote-access. The printed URL contains a generated token. Remote access cannot be combined with --unsafe-unredacted.

Streaming and reconciliation

Snapshots include a monotonically increasing cursor. SSE frames use the same cursor as their id and have types event, finding, source, or heartbeat.

id: 42
event: event
data: {"event_id":"evt-42", ...}

On reconnect, fetch the authoritative run snapshot again, replace local projection state, reconcile event identities by event_id, then subscribe after the returned cursor. A slow or disconnected client never blocks source ingestion.

Sequence message contract

A handoff arrow is emitted when:

  • kind is exactly message.sent.
  • attributes.to is a string naming the destination actor ID.
{
  "kind": "message.sent",
  "actor": {"id": "planner"},
  "attributes": {"to": "researcher"}
}

If the target actor never appears, the sequence view renders an unresolved endpoint. Message links do not alter the primary spawn tree.

Usage and cost contract

Usage values are event-local deltas, never cumulative snapshots. Supported keys are input_tokens, output_tokens, total_tokens, and cost_usd.

They may appear in top-level usage, attributes.usage, top-level individual keys, or individual attribute keys. More structured usage objects take precedence.

"usage": {
  "input_tokens": 120,
  "output_tokens": 40,
  "total_tokens": 160,
  "cost_usd": 0.0042
}

Missing metrics are projected as {"available": false, "value": null}, which is distinct from an observed zero.

Actor role and model contract

actor.id is the stable logical agent invocation identity within a trace. Put a human-facing role in actor.role. Put model identity in actor.model or attributes.model.

The primary parent is the first causal cross-actor parent-span relationship that introduces an actor. Later parent candidates remain causal links and produce an AMBIGUOUS_PARENT projection warning.

Warnings and findings

Runtime warnings come from the core index: LOOP, RETRY, STALL, ORPHAN, and EVICT. Serve mode tracks observed active/resolved state and detection/resolution times during the process lifetime.

Ingestion findings include invalid JSON, invalid events, and duplicate event IDs. Source findings include replacement, truncation, unavailability, and decoding errors. Late events after terminal trace events are instrumentation findings.

Loop, retry, stall, and orphan warnings are debugging heuristics, not guaranteed diagnoses.

Payload and redaction boundary

Sanitization runs before indexing and every browser-visible projection. Ordinary snapshots contain a bounded sanitized payload preview. The lazy payload endpoint contains the retained sanitized full payload only while the server memory budget still retains that payload.

Sensitive keys, common credential formats, bearer values, and structural identifiers follow the same rules as the CLI and TUI. Rejected input diagnostics remain redacted even when unsafe mode is requested.
